# 3.4 Identify Privacy Requirements

## Learning Objectives

- Define data collection scope and minimization principles
- Differentiate between anonymization, pseudo-anonymization, and tokenization
- Describe user rights under major privacy regulations
- Explain data retention requirements and best practices
- Identify cross-border data transfer requirements

---

## Data Collection Scope

Privacy begins with **limiting what data is collected**. The principle of **data minimization** states that organizations should collect only the data necessary for a specific, defined purpose.

| Principle | Description |
|-----------|-------------|
| **Purpose limitation** | Data collected for specified, explicit, and legitimate purposes only |
| **Data minimization** | Collect only what is necessary — one of the most powerful privacy-enhancing technologies |
| **Storage limitation** | Retain data only as long as necessary for the specified purpose |
| **Accuracy** | Keep data accurate and up to date |
| **Lawfulness** | Ensure a legal basis exists for processing personal data |

## Data Anonymization and Privacy-Enhancing Technologies (PET)

### Anonymization vs. Pseudo-Anonymization

| Technique | Reversible? | Description |
|-----------|:-----------:|-------------|
| **Anonymization** | **No** | Permanently removes identifiers; data **cannot** be linked back to an individual |
| **Pseudo-Anonymization** | **Yes** | Replaces identifiers with pseudonyms; data **can** be linked back with additional information |

> **Exam Tip**: Anonymized data is **no longer personal data** under most privacy laws. Pseudo-anonymized data is still considered personal data because it can be reversed.

### Anonymization Techniques

| Technique | Description | Example |
|-----------|-------------|---------|
| **Replacement / Substitution** | Identifiable information replaced with non-identifiable values | Replace name with "Person A" |
| **Suppression / Omission** | Identifiable information is omitted entirely | Remove SSN column from dataset |
| **Generalization** | Specific information replaced with more general form | Replace date of birth with year only |
| **Perturbation / Randomization** | Random changes applied to data values | Add random noise to age values |
| **Scrambling** | Deleting or obfuscating confidential data | Shuffle characters in a field |

### Other Privacy-Enhancing Technologies

| Technology | Description | Reversible? |
|------------|-------------|:-----------:|
| **Data Masking** | Substituting altered values to create a mirror version (e.g., `****1234`) | Yes (a form of pseudo-anonymization) |
| **Tokenization** | Replacing sensitive data with random tokens that can be traced back to original values | Yes |
| **Data Minimization** | Not collecting or storing unnecessary data | N/A |

> **Key Distinction**: **Tokenization** assigns a random value that **can be reversed** to retrieve original data. **Anonymization** techniques like suppression and generalization are **irreversible**.

## User Rights

Major privacy regulations grant individuals specific rights over their personal data:

### Core User Rights (GDPR Framework)

| Right | Description |
|-------|-------------|
| **Right to Information** | Be informed about how personal data is collected and used |
| **Right to Access** | Obtain a copy of personal data being processed |
| **Right to Rectification** | Correct inaccurate or incomplete personal data |
| **Right to Erasure** ("Right to be Forgotten") | Request deletion of personal data when no longer necessary |
| **Right to Restrict Processing** | Request limitation of how personal data is processed |
| **Right to Object** | Object to processing based on legitimate interests or direct marketing |
| **Right to Data Portability** | Receive personal data in a structured, machine-readable format |

### User Consent

| Consent Type | Description |
|-------------|-------------|
| **Informed/Affirmative consent** | User actively agrees to data processing (opt-in) |
| **Explicit consent** | Required for processing special categories of data (health, biometric, etc.) |
| **Parental consent** | Required for children's data (COPPA: under 13; GDPR: varies by member state) |
| **Withdrawal** | Data subject can **withdraw consent at any time** |

### Breach Notification

Organizations must notify individuals and authorities when a data breach occurs:

| Regulation | Notification Timeline |
|-----------|----------------------|
| **GDPR** | 72 hours to supervisory authority; without undue delay to data subjects |
| **HIPAA** | 60 days to affected individuals |
| **State laws (US)** | Typically 30–90 days |

## Data Retention

Data retention defines **how long, where, and what** data is kept.

| Element | Consideration |
|---------|--------------|
| **Duration** | How long must data be retained? (Regulatory minimums and business needs) |
| **Location** | Where is data stored? (Jurisdictional implications) |
| **Content** | What specific data elements are retained? |
| **Access** | Who can access retained data during the retention period? |
| **Disposal** | How is data destroyed when the retention period expires? |

### Common Retention Requirements

| Regulation | Retention Period |
|-----------|-----------------|
| **SOX** | Financial records: 7 years |
| **HIPAA** | Medical records: 6 years from date of creation |
| **PCI DSS** | Audit logs: 1 year (minimum 3 months immediately accessible) |
| **GDPR** | No specific period — "no longer than necessary" for the stated purpose |

## Cross-Border Requirements

When data crosses national boundaries, additional requirements apply:

### Key Concepts

| Concept | Description |
|---------|-------------|
| **Data Residency** | Requirement that data be stored within a specific geographic location |
| **Jurisdiction** | Legal authority over data based on its location or the location of the data subject |
| **Multi-national boundaries** | Data processing that spans multiple countries with different privacy laws |

### GDPR Cross-Border Mechanisms

| Mechanism | Description |
|-----------|-------------|
| **Binding Corporate Rules (BCRs)** | Internal policies for intra-group data transfers |
| **Standard Contractual Clauses (SCCs)** | Pre-approved contract templates for data transfers (also called Model Contract Clauses / MCCs) |
| **Adequacy decisions** | EU Commission determination that a non-EU country provides adequate data protection |

### Other Privacy Regulations

| Regulation | Jurisdiction | Key Feature |
|-----------|-------------|-------------|
| **GDPR** | EU/EEA | Comprehensive data protection with extraterritorial reach |
| **CCPA** (AB 375) | California, USA | Consumer right to know, delete, and opt-out |
| **LGPD** | Brazil | GDPR-similar framework |
| **POPIA** | South Africa | Protection of personal information |

---

## Exam Focus Points

1. **Data minimization**: Most powerful PET — don't collect what you don't need
2. **Anonymization vs. pseudo-anonymization**: Anonymization = irreversible; pseudo-anonymization = reversible
3. **Tokenization**: Reversible replacement with random tokens
4. **Right to be forgotten**: User can request deletion of personal data
5. **GDPR breach notification**: 72 hours to supervisory authority
6. **Cross-border mechanisms**: BCRs, SCCs, adequacy decisions
7. **Consent withdrawal**: Data subject can withdraw consent at any time
8. **Data residency**: Some jurisdictions require data to remain within borders

## Key Terms Glossary

| Term | Definition |
|------|-----------|
| **Data Minimization** | Collecting only the minimum data necessary for a defined purpose |
| **Anonymization** | Irreversible removal of identifying information from data |
| **Pseudo-Anonymization** | Reversible de-identification using pseudonyms |
| **Tokenization** | Replacing sensitive data with reversible random tokens |
| **Data Masking** | Substituting altered values to hide original data (e.g., `****1234`) |
| **Right to Erasure** | Data subject's right to request deletion of personal data |
| **Data Residency** | Requirement to store data within a specific geographic location |
| **BCR** | Binding Corporate Rules for intra-group international data transfers |
| **SCC** | Standard Contractual Clauses for international data transfers |
| **PET** | Privacy-Enhancing Technology |
| **GDPR** | General Data Protection Regulation (EU) |
| **CCPA** | California Consumer Privacy Act |
