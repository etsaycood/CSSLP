# 7.7 Threat Hunting and Intelligence

## Learning Objectives

- Define the purpose and methodology of Threat Hunting
- Differentiate between Reactive Monitoring (SIEM) and Proactive Hunting
- Explain the role of Indicators of Compromise (IoCs)
- Describe the Cyber Kill Chain

---

## Reactive Monitoring vs. Proactive Threat Hunting

Traditional SOC (Security Operations Center) monitoring relies on the SIEM. If a known bad IP addresses hits the firewall, or an antivirus catches malware, an alert fires. The analyst responds. **This is reactive.**

What if the attacker is employing a "living off the land" technique, using legitimate administrative tools (like PowerShell or WMI) without using any known malware? The SIEM might not alert because nothing looks inherently "bad."

**Threat Hunting is proactive.** It assumes the attacker is *already inside the network* and has bypassed traditional defenses.

| Capability | SIEM Monitoring | Threat Hunting |
|------------|-----------------|----------------|
| **Stance** | Reactive (Wait for an alert). | Proactive (Actively search). |
| **Trigger** | A signature matched a known threat. | A human-driven hypothesis based on intelligence or anomalies. |
| **Focus** | Known bads (CVEs, malware hashes). | Unknown unknowns, subtle behavioral anomalies. |
| **Automation** | Highly automated. | Highly manual, requiring deep human expertise. |

---

## The Threat Hunting Methodology

Hunting is an iterative process driven by the scientific method.

1. **Form a Hypothesis**: "Based on a new threat intelligence report about the APT29 group, I hypothesize they might be using malicious Office macros to download PowerShell payloads on our HR systems."
2. **Investigate details**: The hunter queries the SIEM and endpoint logs for specific behaviors (e.g., instances where MS Word spawned a PowerShell process).
3. **Uncover new patterns**: The hunter realizes that PowerShell was indeed executed, communicating with an unknown external IP address on port 443.
4. **Enrich Analytics**: The hunter takes this newly discovered, undocumented IP address and feeds it back into the SIEM and Firewall as a new block rule.
5. **Respond**: If a breach is confirmed, declare an Incident and transition to the PICERL Incident Response lifecycle.

## Threat Intelligence and IoCs

Threat hunting is fueled by Threat Intelligence—data about attackers' motives, targets, and behaviors.

### Indicators of Compromise (IoCs)
IoCs are the forensic artifacts left behind by an intrusion. Hunters search for IoCs.

| Type | Example | Usefulness |
|------|---------|------------|
| **Hashes** | SHA256 of a malware file. | Trivial for attackers to change (recompile malware). Weak indicator. |
| **IP Addresses** | Command and Control (C2) server IP. | Easy for attackers to change using cloud providers or proxies. |
| **Domain Names** | `evil-hacker.com` | Requires DNS registration. Harder to change, but still manageable. |
| **Network Artifacts** | Specific User-Agent strings or HTTP header anomalies. | Good indicator if the attacker uses custom tools. |
| **Tools** | Finding Mimikatz or Cobalt Strike on a server. | Strong indicator of a skilled attacker. |
| **TTPs (Tactics, Techniques, and Procedures)** | Using Scheduled Tasks for persistence, or specific PowerShell execution policies. | **The strongest indicator.** Even if they change IP addresses and domains, their *behavior* remains similar. Focus hunting on TTPs. |

## The Cyber Kill Chain (Lockheed Martin)

Understanding the phases of an attack helps hunters identify where to look for IoCs. If you break the chain at any point, the attack fails.

1. **Reconnaissance**: Harvesting email addresses, scanning public IPs. (Hard to detect).
2. **Weaponization**: Creating the payload (e.g., a PDF with an exploit).
3. **Delivery**: Emailing the PDF to the victim. (Hunt for phishing emails).
4. **Exploitation**: The PDF executes code on the victim's OS. (Hunt for crash logs or buffer overflows).
5. **Installation**: Installing a backdoor for persistence. (Hunt for new registry keys, scheduled tasks, or abnormal services).
6. **Command and Control (C2)**: The backdoor beacons out to the attacker's server for instructions. (Hunt for strange outbound DNS requests or steady, periodic beaconing traffic).
7. **Actions on Objectives**: Exfiltrating the gigabytes of database records to a cloud storage site. (Hunt for massive outbound data spikes during off-hours).

---

## Exam Focus Points

1. **Hunting is Proactive**: It assumes the network is already breached and defenses have failed.
2. **Hypothesis-Driven**: Hunts don't start blindly; they begin with an educated guess based on threat intel.
3. **TTPs > Hashes**: Focus hunting on attacker behavior (TTPs), not just easily changed characteristics like IP addresses or file hashes.
4. **The Kill Chain**: Defending successfully requires breaking the chain before "Actions on Objectives" is reached.

## Key Terms Glossary

| Term | Definition |
|------|-----------|
| **Threat Hunting** | Proactive search for cyber threats that are undetected in a network |
| **IoC** | Indicator of Compromise; forensic evidence of a breach |
| **TTP** | Tactics, Techniques, and Procedures; the behavioral patterns of an attacker |
| **Cyber Kill Chain** | A framework outlining the stages of a cyberattack |
| **C2 (Command and Control)** | Infrastructure an attacker uses to maintain communication with compromised systems |
