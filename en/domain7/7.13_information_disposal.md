# 7.13 Information Disposal and Media Sanitization

## Learning Objectives

- State the critical risks of improper data disposal
- Differentiate between Clearing, Purging, and Destruction
- Explain Degaussing and Crypto-Shredding techniques
- Apply the appropriate data disposal method based on media type (HDD, SSD, Cloud)

---

## The Danger of Data Remanence

When a user hits "Delete" or formats a hard drive in an operating system, the data is not truly erased. The OS merely deletes the *pointer* to the file, marking that sector of the hard drive as available for overwriting. Until new data actively overwrites that specific sector, forensic tools can easily recover the "deleted" data.

This phenomenon is called **Data Remanence**. To securely dispose of information, you must actively sanitize the media so recovery becomes mathematically impossible.

---

## Levels of Data Sanitization (NIST SP 800-88)

NIST defines three categories of sanitization, escalating in effectiveness and cost.

### 1. Clearing
- **Definition**: Removing data using logical software techniques so that it cannot be recovered using standard system tools.
- **Method**: Writing zeroes or random characters across the entire drive (e.g., a "Zero-fill" or using tools like `/dev/urandom` in Linux).
- **Use Case**: Re-provisioning a laptop for a new employee internally within the same company.
- **Limitation**: Advanced laboratory techniques (magnetic force microscopy) might theoretically recover data, especially on traditional magnetic hard drives.

### 2. Purging
- **Definition**: Removing data so that it cannot be recovered using state-of-the-art laboratory techniques.
- **Methods**:
    - **Degaussing**: Using a massive electromagnetic pulse to instantly scramble the magnetic domains on a platter. (Destroys the hard drive permanently).
    - **Secure Erase (ATA SE)**: A firmware-level command built into modern drives that forces the controller to overwrite all sectors.
- **Use Case**: Disposing of hard drives or selling them outside the organization.

### 3. Destruction
- **Definition**: The ultimate form of sanitization. The physical destruction of the media itself.
- **Methods**: Shredding (industrial woodchippers for metal), pulverization, incineration, acid baths.
- **Use Case**: Highly Classified government data, or situations where the drive has failed mechanically and cannot be logically purged or degaussed.

---

## Challenges with Solid State Drives (SSDs)

Traditional methods like clearing (overwriting) and degaussing **do not work predictably on SSDs or USB flash drives**.

- **Why Overwriting Fails**: SSDs use "Wear Leveling" algorithms. The controller constantly moves data around in the background to prevent specific memory cells from burning out. If you tell the OS to overwrite a file, the controller might just write the zeroes to *new* cells, leaving the original data perfectly intact in the old cells.
- **Why Degaussing Fails**: SSDs store data using trapped electrical charges in transistors, not magnetic fields. Therefore, a magnet has absolutely no effect on an SSD.

**How to destroy an SSD?**
1. Physical shredding/pulverization into tiny pieces (2mm or smaller).
2. Manufacturer-provided firmware Secure Erase (ATA SE) commands, which forcefully flush all electrons from the NAND flash cells.

## Crypto-Shredding (Cloud Sanitization)

If your database is hosted in AWS, Azure, or a SaaS provider like Salesforce, you cannot fly to their datacenter, locate the specific hard drive containing your data (in a multi-tenant array), and toss it into an incinerator. 

How do you securely dispose of data you cannot physically touch?

### Cryptographic Erasure (Crypto-Shredding)
1. **Rule**: All sensitive data stored in the cloud must be encrypted *before* it is written to the disks. The organization stores the encryption key in a separate, highly secure central KMS (Key Management Service).
2. **Disposal**: When you want to permanently destroy the 100 Terabytes of backup data stored in the cloud, you simply tell your KMS to **delete the encryption key**.
3. **Result**: The 100TB of data sitting on Amazon's hard drives instantly becomes permanently unreadable mathematically random noise. It is effectively "destroyed" in milliseconds without needing to overwrite the data or touch the physical hardware.

---

## Verification and Certification

A critical final step in the decommissioning process is obtaining a **Certificate of Destruction**.
- If you hire an external vendor to shred hard drives, they must provide a legally binding document detailing the serial number of every drive destroyed, the date, and the method.
- This certificate is required by auditors (PCI DSS, HIPAA) to prove compliance.

---

## Exam Focus Points

1. **Delete ≠ Erased (Data Remanence)**: Deleting simply removes the pointer. Overwriting is required for logical sanitization.
2. **Degaussing limitations**: Magnets destroy traditional Hard Disk Drives (HDDs) and magnetic tapes. Degaussers are completely useless against Solid State Drives (SSDs) and optical media (CDs/DVDs).
3. **Crypto-Shredding**: The only viable, mathematically secure method for sanitizing data in the Cloud. You encrypt the data, then destroy the key.
4. **Wear Leveling**: The feature in SSDs that makes standard logical overwriting highly unreliable.

---

## Key Terms Glossary

| Term | Definition |
|------|-----------|
| **Data Remanence** | The residual physical representation of data that has been nominally erased |
| **Clearing** | Overwriting data to prevent basic software recovery |
| **Purging** | Advanced sanitization (like degaussing) to defeat laboratory recovery techniques |
| **Degaussing** | Using strong magnetic fields to scramble magnetic media (HDDs, tapes) |
| **Crypto-Shredding** | Rendering encrypted data unreadable by deliberately destroying the decryption key |
| **Wear Leveling** | SSD technology that evenly distributes writes to prolong drive life, complicating secure erasure |
