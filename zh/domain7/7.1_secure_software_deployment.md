# 7.1 執行軟體安全部署 (Perform Secure Software Deployment)

## 學習目標

- 定義軟體安全發佈與部署的必備先決條件與要求
- 解釋各種不同的部署策略 (藍綠部署 Blue/Green、金絲雀發佈 Canary)
- 描述開機啟動序列 (如 Secure Boot 安全開機) 在資安防禦上的極端重要性
- 解釋「基礎設施即程式碼 (IaC)」在實現安全部署市場所扮演的關鍵角色

## 安全發佈前的準備工作 (Secure Release Preparation)

「部署 (Deployment)」是軟體從受到嚴密保護的溫室測試環境，正式踏入充滿敵意與猛獸的 Production 真實戰場的關鍵時刻。這個跨越生死線的動作，絕對需要最嚴謹的治理與把關。

### 營運授權放行 (Authorization to Operate, ATO)
在部署按鈕被按下之前，系統必須先經歷一場嚴格的**「評估與授權 (A&A, Assessment and Authorization)」**儀式。
- 這場儀式的核心人物是**資料擁有者 (Data Owner) / 資訊擁有者 (Information Owner)**。他們會滿臉嚴肅地審閱那份記載著「殘餘風險 (residual risk)」的期末報告。
- 如果他們認為這些風險在長官能承受的範圍內，他們就會大筆一揮，正式頒發一張**「營運授權 (ATO)」**的免死金牌。
- **沒有拿到 ATO 就膽敢私自把系統推上線，是會被直接革職的嚴重合規性天條 (compliance violation)**。

### 部署前夕的生死檢核表 (Pre-Deployment Checklist)
- [ ] 程式碼簽章 (Code signing) 已完成。(這能確保準備上線的二進位檔案，在運送途中沒有被掉包或竄改)。
- [ ] SAST / DAST / SCA 所有的漏洞掃描都已經亮綠燈通過。
- [ ] 滲透測試報告裡被揪出來的致命漏洞，都已經被確認修復補好。
- [ ] 組態設定基準 (Configuration baseline) 已經確立，且伺服器已完成所有安全強固貼片 (hardened)。
- [ ] **退版復原計畫 (Rollback Plan)** 已經寫好，而且**真的實際演練測試過**。

> **考試提示 (Exam Tip)**：考卷上的每一套部署計畫，**「絕對」都要綁定一份經過測試的退版復原計畫 (Rollback Plan)**。如果這次上線大翻車，或是引發了災難級的資安漏洞，你的團隊必須有能力在最短的時間內，神奇地把系統「倒轉」回昨天那個美好且安全的舊版本狀態。

## 部署戰略大觀園 (Deployment Strategies)

現代崇尚敏捷開發的 IT 環境中，我們早已揚棄那種「半夜停機三小時慢慢更新」的土法煉鋼；取而代之的是那些能把停機時間逼近於零、且能將災情爆炸半徑 (blast radius) 縮到極致的優雅部署戰略。

| 戰略名稱 | 戰術說明 | 資安與維運上的巨大紅利 (Security Benefit) |
|----------|-------------|------------------|
| **藍綠部署 (Blue/Green Deployment)** | 準備兩座一模一樣的 Production 環境 (Blue 正在服役接客，Green 在旁邊喝茶納涼待命)。先把最新版的程式碼部署到 Green 上，在內部測試沒問題後；只消動一下滑鼠切換 Load Balancer/Router 的指向，瞬間把所有客人導向 Green。 | **擁有「秒級」退版復原能力 (Instant rollback)** (如果 Green 大翻車，滑鼠點一下把路由切回 Blue 就天下太平了)。而且能做到真正的「零停機時間 (Zero downtime)」。 |
| **金絲雀發佈 (Canary Release)** | 只把最新版的程式碼，悄悄發佈給一小群白老鼠用戶 (例如：隨機挑 5% 的流量)。然後死盯著錯誤報表跟資安警報。如果這 5% 的金絲雀沒被毒死 (運作穩定)，再慢慢把流量推廣至 100%。 | **完美限縮了災情的「爆炸半徑 (Limits the blast radius)」**。萬一新版本裡藏著一個會毀滅世界的致命漏洞，死傷範圍也只會侷限在那倒霉的 5% 用戶身上而已。 |
| **滾動式部署 (Rolling Deployment)** | 像貪吃蛇一樣，在伺服器叢集中一台接著一台、緩慢地把舊版本抽換成新版本。 | 在部署期間依然能維持系統的高可用性；但它的致命傷是：如果發現不對勁想退版 (rollback)，速度會比藍綠部署慢上非常多。 |
| **功能開關 (Feature Toggles / Flags)** | 把新功能的程式碼默默先推上 Production，但用一個「設定檔裡的開關 (flag)」把它藏起來不讓任何人看見。等到黃道吉日，只要改個設定檔把開關「On」打開，新功能就會瞬間降臨。 | **帶來極致光速的退版體驗 (Extremely fast rollback)**。只要發現新功能狂噴錯誤，只需把開關「Off」切掉就好，完全不需要勞師動眾再去重新部署一次舊版程式碼。 |

## 作業環境的開機護城河 (Environment Boot Security)

光是把應用軟體寫得再安全也沒用；如果承載它的作業系統 (OS) 骨子裡在開機的那一刻，就已經被藏在底層的 Rootkit (惡意核心套件) 給綁架了，那你所有的上層防禦都只是紙老虎。

### 安全開機與量測啟動 (Secure Boot and Measured Boot)

| 護城河技術 | 防禦原理說明 |
|------------|-------------|
| **安全開機 (Secure Boot)** | 這是一項寫在主機板 UEFI 裡的防線。它嚴格規定：電腦在開機時，**只准載入那些「擁有硬體原廠數位簽章、被認證為絕對清白」的開機管理程式 (bootloaders) 與作業系統核心 (OS kernels)**。這能完美防堵 Rootkit 搶在 OS 啟動前先一步劫持系統。 |
| **量測啟動 (Measured Boot)** | 在系統一層層啟動的過程中，它會冷酷地幫每一個被載入的開機元件「計算雜湊值 (Hashes)」，並把這些鐵證死死地鎖進主機板上的 TPM 晶片裡。遠端的伺服器稍後可以透過「遠端證明 (Remote Attestation)」來驗證這些雜湊值，以確認這台機器從硬體到核心有沒有被動過手腳。 |
| **TPM 晶片** | 全名Trusted Platform Module (可信賴平台模組)。它是焊死在主機板上的一顆專用硬體安全晶片；專職負責安全地生成、儲存那些命根子般的密碼編譯金鑰、密碼與數位憑證。 |

## 基礎設施即程式碼 (IaC, Infrastructure as Code)

在現代化的雲端部署戰場上，幾乎不會再有人類工程師用 SSH 登入伺服器，在那邊一行行敲指令安裝軟體了。取而代之的是，**連同整個基礎設施的長相 (幾台機器、防火牆怎麼開)，全都被寫成了一行行的程式碼** (例如：Terraform, Ansible, AWS CloudFormation)。

### IaC 為資安帶來的革命性紅利 (Security Benefits of IaC)
- **極致的穩定與一致性 (Consistency)**：徹底消滅了傳說中的**「組態漂移 (configuration drift)」** (也就是伺服器用久了，因為工程師天天在那手動亂改設定，導致最後每台伺服器的長相都不一樣的慘劇)。
- **不可變更的基石 (Immutability)**：伺服器升格為免洗筷，我們「替換它，而不是修補它 (replaced, not patched)」。如果某台伺服器疑似中毒或被攻陷，我們不會派人上去抓毒；我們直接把它殺了 (destroyed)，然後用乾淨無瑕的 IaC 模板在一秒內重新變出一台全新的。
- **資安左移的極致展現 (Shift-Left Security)**：這些定義防火牆與機器的 IaC 腳本檔案，在被拿去正式架設機器**之前**，就能先送進 SAST 工具裡掃描。這能在機器「還沒被生出來之前」，就把像是「S3 Bucket 全世界都可讀取」這種致命的設定失誤給攔截下來。
- **完美的稽核軌跡 (Auditability)**：因為基礎設施現在全變成了「程式碼」，所以對防火牆或架構做的每一次異動，都會被 Version Control (如 Git) 給死死記錄下來。它為稽核員提供了一條無懈可擊的證據鏈：誰、在幾月幾號幾點、為了什麼原因、去偷動了哪一條防火牆規則。

## 考試重點

1. **退版復原計畫 (Rollback Plan)**：每次部署的標準配備。如果沒有一條經過驗證的退路，你連部署按鈕都不准碰。
2. **ATO (營運授權)**：長官 (資料/系統擁有者) 點頭放行的簽字。沒有 ATO 就硬把軟體推上線，在商用環境是大忌。
3. **藍綠部署 vs. 金絲雀發佈**：打法不同。藍綠部署主打「零停機瞬間退版」；金絲雀發佈則是用來「控制未知蟲子咬人的爆炸半徑」。
4. **安全開機 (Secure Boot) 是殺 Rootkit 的剋星**：它透過檢查數位簽章，確保開機時只有根正苗紅的 OS 核心可以被喚醒。
5. **IaC 徹底消滅了「組態漂移 (Config Drift)」**：把機器架構當成程式碼來管理，能確保機房裡的一千台伺服器，長得跟設計草圖裡要求的一模一樣分毫不差，而且全都經過了最嚴格的安全強固。

## 關鍵術語表

| 術語 | 定義 |
|------|-----------|
| **ATO** | Authorization to Operate (營運授權)；這是由管理高層正式簽發、准許系統上線運作的終極許可證 |
| **Blue/Green Deployment (藍綠部署)** | 準備兩套完全一樣的環境，讓你在新舊版本間進行「零停機切換」與「瞬間退版」的神仙策略 |
| **Canary Release (金絲雀發佈)** | 拿一小撮真實用戶當白老鼠先試用新版本，藉此在全面鋪開前先測試環境穩定度的風險控管手法 |
| **Secure Boot (安全開機)** | 一項主機板 UEFI 提供的高級護城河功能；它會嚴格檢查開機管理程式的數位簽章，用來防堵底層的惡意軟體 |
| **TPM** | Trusted Platform Module (可信賴平台模組)；主機板上的硬體保險箱，專門用來存放最重要的加密金鑰與憑證 |
| **IaC** | Infrastructure as Code (基礎設施即程式碼)；透過撰寫給機器讀懂的定義檔腳本，來全自動化部署並配置雲端運算資源的作法 |
