# 7.7 威脅獵捕與資安情報 (Threat Hunting and Intelligence)

## 學習目標

- 定義「威脅獵捕 (Threat Hunting)」的核心目的與執行方法論
- 將傳統的「被動式監控 (SIEM)」與「主動出擊的威脅獵捕」進行強烈且清晰的對比區分
- 解釋「妥協指標 / 威脅指標 (IoCs, Indicators of Compromise)」在辦案中所扮演的關鍵角色
- 描述洛克希德·馬丁公司提出的「網路殺傷鏈 (Cyber Kill Chain)」七大階段

---

## 守株待兔 vs. 主動出擊 (Reactive Monitoring vs. Proactive Threat Hunting)

傳統資安戰情室 (SOC) 的運作模式，幾乎完全依賴 SIEM 的「被動監控」。流程是這樣的：如果有個惡名昭彰的壞 IP 撞了防火牆，或是防毒軟體比對到一隻病毒的特徵碼，警報就會尖叫。接著，分析師才開始起床動手處理。**這叫作「被動反應式防禦 (Reactive)」。**

但問題來了：如果今天這個狡猾的駭客根本不用自己帶木馬病毒，而是直接就地取材（這叫 **「離地攻擊 / Living off the land」**），直接徵用系統內建合法、乾淨的管理員指令 (例如 PowerShell 或是 WMI) 來幹壞事呢？這時候笨笨的 SIEM 根本不會拉警報，因為在它眼裡，這些指令看起來就像是普通的 IT 管理員在加班做事而已。

**這時候，我們就需要「主動出擊的威脅獵捕 (Threat Hunting)」。** 獵捕行動的核心信仰只有一個：**「永遠假設防線早就被攻破了，駭客現在已經藏在我們公司內網的某個角落。」**

| 戰力比較 | 傳統 SIEM 監控 (SIEM Monitoring) | 威脅獵捕 (Threat Hunting) |
|------------|-----------------|----------------|
| **防禦姿態 (Stance)** | 被動 (Reactive)。乖乖坐在椅子上等警報響。 | 主動 (Proactive)。帶槍走出戰情室去叢林裡找壞人。 |
| **啟動條件 (Trigger)** | 剛好有一條現成的特徵碼 (signature) 吻合了。 | 來自獵人腦袋裡的直覺，或是憑藉最新情資做出的「大膽假設 (hypothesis)」。 |
| **關注焦點 (Focus)** | 已知的舊敵人 (已知病毒的 Hashes、已知的 CVEs)。 | 「未知的未知 (Unknown unknowns)」，以及藏在正常行為背後的詭異微細波動。 |
| **自動化程度 (Automation)** | 高度自動化，全靠機器判讀。 | 極度依賴人工，需要資安老鳥極深厚的直覺與技術底子。 |

## 獵人的狩獵法則論 (The Threat Hunting Methodology)

威脅獵捕不是像無頭蒼蠅一樣在日誌海裡亂翻；它是一套嚴謹的、奠基於「科學方法」的疊代循環流程。

1. **大膽假設 (Form a Hypothesis)**：「我剛看完一篇關於 APT29 駭客集團的最新情資報告。我*假設*這群人現在可能正潛伏在我們公司的 HR 部門裡，試圖用夾帶惡意巨集的 Word 檔案來偷偷下載 PowerShell 後門。」
2. **深入叢林調查 (Investigate details)**：獵人開始敲鍵盤，調閱 SIEM 紀錄與 HR 員工電腦的日誌，尋找破案線索（例如：專門去搜把「有沒有發生過微軟 Word 軟體突然莫名其妙呼叫出 PowerShell 終端機」的靈異事件）。
3. **拼湊出犯罪模式 (Uncover new patterns)**：獵人眼睛一亮！他發現 PowerShell 還真的被叫出來了，而且它還趁沒人注意時，偷偷連線到了一個網路上從沒看過的神秘 IP (而且還是走看似合法的 Port 443 加密通道)。
4. **豐富情報武器庫 (Enrich Analytics)**：獵人把這個這輩子沒見過的新發現 (未知 IP 地址)，立刻寫進 SIEM 的規則裡，並下令防火牆馬上全面封殺這個 IP。你的防護網因為這場狩獵而升級了！
5. **切換身分應戰 (Respond)**：既然已經抓到駭客的尾巴，確認這不是演習，那就立刻通報長官宣布進入「Incident (資安事件)」，所有人放下手邊工作，全速切換到上一章教過的 PICERL 事件應變生命週期 (立刻開始進行控制止血 Containment)。

## 威脅情資與辦案線索 IoC (Threat Intelligence and IoCs)

如果沒有外在的「威脅情資 (Threat Intelligence)」告訴獵人現在外面流行什麼最新殺手、愛用什麼武器，獵捕行動就會淪為瞎子摸象。

### 威脅指標 / 妥協指標 (IoCs, Indicators of Compromise)
IoCs 就是駭客在犯罪現場不小心遺留下來的「指紋、腳印或彈殼」。獵人每天上班就是在尋找這些 IoCs。

但這些證據的價值，有著天壤之別 (這就是所謂的"痛苦金字塔 Pyramid of Pain")：

| 證據類型 | 辦案範例 (Example) | 它的防禦價值有多高？ (Usefulness) |
|------|---------|------------|
| **檔案雜湊值 (Hashes)** | 惡意軟體檔案的 SHA256 指紋。 | **(弱彈)** 駭客只要隨便改個程式碼重新編譯 (recompile)，雜湊值就會瞬間變了。這根本防不住他們。 |
| **IP 地址 (IP Addresses)** | 駭客用來遙控木馬的 C2 伺服器 IP。 | 駭客只要去 AWS 或租個海外跳板 Proxy 就能輕鬆換掉 IP。 |
| **網域名稱 (Domain Names)** | `evil-hacker.com` | 要架設這個需要花點錢去註冊。稍微提高了一點駭客變身的成本，但還是不難。 |
| **網路傳輸特徵 (Network Artifacts)** | 封包裡面夾著奇怪的 User-Agent 標籤，或是 HTTP 標頭檔裡有不尋常的拼字錯誤。 | **(好物)** 如果駭客是用自己手刻的奇葩工具，這種網路傳輸上留下的個人癖好，是很好的追蹤線索。 |
| **工具箱 (Tools)** | 在公司伺服器裡搜出 Mimikatz (超強密碼抓取神器) 或是 Cobalt Strike 的蹤跡。 | 代表你們家已經被具備一定實力的頂尖職業級駭客盯上了。 |
| **TTPs (戰術、技術與作業程序)** | 駭客特別喜歡用「Windows 排程工作」來做後門持久化，或是他對操控特定頻率的 PowerShell 參數有著謎之執著。 | **(最強的終極核彈)** 這是駭客根深蒂固的**「行為模式與作案手法」**。就算他們換了一百個 IP 和病毒檔案，他們寫扣跟思考的拿手把戲 (TTPs) 短時間內絕對改不掉。**高階的威脅獵捕，完全聚焦在獵殺 TTPs 上。** |

## 網路殺傷鏈模型 (The Cyber Kill Chain / 由 Lockheed Martin 提出)

要想抓賊，你得先了解賊是怎麼破門的。洛克希德·馬丁公司把一場完美的網路攻擊，拆解成了極具破壞美學的七階段「殺傷鏈」。
**防禦者的絕對優勢：只要你能在此鏈條的「任何一個環節」成功把鏈條打斷，駭客的心血就會全部白費，攻擊宣告破局。**

1. **偵察探路 (Reconnaissance)**：駭客躲在暗處蒐集你們公司高管的 E-mail 信箱，用掃描器敲打外部防火牆。(最難防範，因為都在系統外面發生)。
2. **武器化準備 (Weaponization)**：駭客在家裡把木馬病毒跟零日漏洞綑綁在一起，塞進一個名為「年度薪資調整.pdf」的假文件中。
3. **散佈投遞 (Delivery)**：駭客透過魚叉式釣魚郵件，把這份附有劇毒的 PDF 寄給你們的人資主管。(防禦點：請獵人在此攔截可疑郵件)。
4. **弱點利用 (Exploitation)**：人資主管點開了 PDF。這份文件瞬間觸發了系統的漏洞，讓惡意代碼在背景悄悄執行。(防禦點：看日誌裡有沒有軟體異常崩潰的紀錄)。
5. **安裝扎根 (Installation)**：病毒開始在電腦裡四處鑽洞，安裝後門程式以確保下次就算電腦重開機，它也還活著。(防禦點：獵捕那些半夜無端被新增的奇怪登錄檔 registry keys 或排程工作)。
6. **指揮與控制 (Command and Control, C2)**：裝好的後門會像是間諜一樣，定時朝著外部的駭客總部 (C2 伺服器) 發射訊號：「長官，我已成功潛伏，請下達指令」。(防禦點：緊盯防火牆，找出那些每十分鐘就規律往國外詭異 IP 發送的神秘脈衝流量 beaconing)。
7. **目標達成行動 (Actions on Objectives)**：遊戲結束。駭客開始大肆把資料庫裡幾十 GB 的客戶資料，趁著半夜打包上傳到國外的雲端硬碟。(防禦點：監控半夜時段不尋常的超長、超巨大的對外資料傳輸暴衝)。

## 考試重點

1. **獵人不等人 (Hunting is Proactive)**：傳統 SIEM 是被動等警報響；威脅獵捕則是「主動出擊」，帶槍親自去雜亂的日誌叢林裡翻找那些 SIEM 根本看不懂的詭異腳印。
2. **由大膽假設驅動的行動 (Hypothesis-Driven)**：獵捕行動絕對不能漫無目的亂搜；出發前，你得先基於一篇最新的「威脅情資情報」，提出一個具有方向性的辦案假說。
3. **TTPs 徹底完爆 Hashes (特徵碼)**：在尋找 IoCs 證據時，去追殺駭客的作案習慣與手法 (**TTPs**) 所能帶來的防禦價值，遠遠甩過只能用一次就被廢掉的檔案雜湊值 (Hashes) 好幾條街。
4. **網路殺傷鏈 (The Kill Chain)**：一場防禦戰要成功，不是非得在第一步就把駭客擋下來；只要能在駭客進行到第七步「把資料偷出去 (Actions on Objectives)」之前，中途切斷任何一個鏈條，這次的防禦就算你贏了。

## 關鍵術語表

| 術語 | 定義 |
|------|-----------|
| **Threat Hunting (威脅獵捕)** | 假設駭客已經突破了所有防火牆潛伏在內網中，並主動派人進入日誌叢林裡將他們揪出來的主動式防禦行動 |
| **IoC (妥協指標/威脅指標)** | Indicator of Compromise；就像案發現場留下的彈殼或血跡，是證明系統已經被駭客入侵的鐵證與線索 |
| **TTP** | Tactics, Techniques, and Procedures (戰術、技術與作業程序)；駭客執行攻擊時其背後的思考邏輯、慣用兵器與行腳習慣，也是最高價值的偵測目標 |
| **Cyber Kill Chain (網路殺傷鏈)** | 一套將駭客攻擊行動拆解為七大階段分析的經典軍事框架模型 |
| **C2 (Command and Control)** | 指揮與控制伺服器；這是駭客蓋在國外的基地台，專門用來遠端操控已經潛伏在你們公司裡的木馬部隊 |
